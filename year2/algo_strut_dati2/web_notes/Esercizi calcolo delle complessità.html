Esercizio 1

Calcoliamo la complessità del seguente algoritmo

A(int n) {
	s = 0
	for i = 1 to n {
		s = s + B(i)
	}
	return s
}

B(int n){
	s = 0
	for j = 1 to n {
		s = s + 1
	}
	return s
}

Conviene calcolare prima la complessità della funzione B, essa ha un ciclo for che viene eseguito n volte e delle istruzioni costanti

TB(n)=Θ(n)T_B(n) = \Theta(n)TB​(n)=Θ(n)

Per quanto riguarda la funzione A, è presente un ciclo eseguito n volte in cui viene chiamata la funzione B, quindi la sua complessità è data dalla somma di n volte la complessità di B, quindi B(1) + B(2) + B(3) + ... + B(n)

TA(n)=∑i=1nΘ(i)T_A(n) = \sum_{i = 1}^n \Theta(i)TA​(n)=i=1∑n​Θ(i)

possiamo scrivere Θ(i)\Theta(i)Θ(i) come i⋅ci \cdot ci⋅c dove

TA(n)=∑i=1n(i⋅c)T_A(n) = \sum_{i = 1}^n (i \cdot c)TA​(n)=i=1∑n​(i⋅c)

TA(n)=c⋅∑i=1niT_A(n) = c\cdot \sum_{i = 1}^n iTA​(n)=c⋅i=1∑n​i

∑i=1ni\sum_{i = 1}^n i∑i=1n​i è una progressione aritmetica che può essere scritta come:

TA(n)=c⋅n⋅(n−1)2T_A(n) = c\cdot \frac{n\cdot (n-1)}{2}TA​(n)=c⋅2n⋅(n−1)​

TA(n)=Θ(n2)T_A(n) = \Theta(n^2)TA​(n)=Θ(n2)

Possiamo realizzare lo stesso scopo di queste due funzioni con una singola funzione di complessità costante

Notiamo che B(n)=nB(n) = nB(n)=n

quindi la funzione A si potrebbe scrivere come

A(int n) {
	s = 0
	for i = 1 to n {
		s = s + i
	}
	return s
}

che rappresenta la somma di tutti i numeri fino ad n, cioè

∑i=1ni=n⋅(n−1)2\sum_{i = 1}^ni = \frac{n \cdot (n-1)}{2}i=1∑n​i=2n⋅(n−1)​

quindi la nuova funzione ottimizzata sarebbe:

A(int n) {
	return n*(n-1)/2
}
